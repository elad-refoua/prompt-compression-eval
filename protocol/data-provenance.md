## 3. Generation Process (Complete Record)

### 3.1 Architecture

**Each conversation was generated by an independent "blind" agent.**

- The orchestrating Claude Code session spawned agents via the Agent tool
- Each agent was a separate, independent process with **no memory of other agents, other conditions, or the study design**
- The agent only knew: its character source file, the therapist's first message, and instructions to play both roles for 10 turns
- Agents ran in parallel batches (up to 8 simultaneously)
- All 20 conversations completed within a ~30-minute generation window

Here are the **exact instructions** sent to each agent. The full text is preserved for replication.

## 4. Methodological Limitations (Full Disclosure)

### 4.1 No Formal Therapist System Prompt

**This is the most important limitation to disclose.**

- The **first** therapist message was fixed and standardized (see `therapist-inputs.md`)
- Subsequent therapist messages (turns 2-10) were generated by the same agent that was playing Sara
- The continuation was guided only by short stylistic directives in the generation prompt (e.g., "Novice continues: advice, closed questions, solution-focused")
- **There was no separate therapist system prompt, no independent therapist agent, no detailed therapist character specification**

**Implications:**
- The therapist's behavior is influenced by the agent's implicit understanding of what a "novice therapist" sounds like
- The therapist is not strictly controlled across conditions -- both agents received the same stylistic directive, but each generates its own continuation
- Therapist consistency within a skill level is an emergent property of the agent's training, not a controlled variable

**Why this design was chosen:**
- Simplicity: one agent per conversation is faster and cheaper
- Consistency with the prior plan: the original MENTI training plan (v6) also used this single-agent approach
- The primary research question is about the CLIENT (Sara), not the therapist -- the therapist is a probe, not the object of study

**How the paper should address this:**
- Disclose openly in the Methods section
- Include the continuation directives verbatim in a supplementary materials file
- Acknowledge that a stronger design would use separate therapist agents (future work)
- Note that the SAME directive was used in both conditions, so any differences in Sara's responses are attributable to her system prompt (33K vs. 3K), not to the therapist

### 4.2 Single-Agent Design

The same AI agent plays both the therapist and the client. This may produce dialogue that is more cooperative than adversarial (the agent may implicitly coordinate both sides). A stronger design would use separate therapist-agent and client-agent instances.

### 4.3 Model Homogeneity

All conversations generated by Claude Opus 4.6. Model-specific biases in language style, clinical reasoning, or Hebrew usage may affect both conditions equally (so comparisons remain valid) but limit generalizability to other AI models.

### 4.4 Temporal Homogeneity

All conversations generated in a single session (2026-04-09 to 2026-04-10). No test-retest of generation stability.

### 4.5 Single Persona

Only Sara was tested. The study cannot speak to other personas (e.g., Noa with borderline personality disorder, Menachem with complicated grief, Yossi the child patient). Generalizability to other clinical presentations is unknown until replication.

### 4.6 Rule Line in 3K Card

The 3K persona card contains one explicit rule line that is NOT in the 33K prompt: "After any opening, regress using a different mechanism each time. Zigzag is mandatory. Never open linearly."

- **Why:** The 33K prompt encodes non-linearity through 15+ scattered references and examples. The 3K card cannot fit these, so the rule is compressed into a single directive.
- **Implication:** The 3K card is not a strict subset of the 33K prompt. It is a compressed encoding with one explicit meta-rule.
- **Disclosure:** This should be mentioned in the Methods section. Readers should know that the 3K card is **not identical to a naive extraction** from the 33K prompt.

### 4.7 No Conversations Were Excluded

All 20 generated conversations were included in the stimulus set. No cherry-picking. No regeneration based on content quality. This is a one-shot generation study.

## 5. Blinding Protocol (Rater Side)

### 5.1 Expert Raters See

Each expert rater sees 10 pairs of conversations. Each pair contains:
- One 33K conversation
- One 3K conversation
- Both using the same therapist input

The conversations are labeled only as "שיחה א'" (Conversation A) and "שיחה ב'" (Conversation B). The assignment of 33K and 3K to A and B is **randomized per pair per expert at runtime** using JavaScript's `Math.random()`.

### 5.2 What the Platform Records

For each pair, the platform records:
- Which condition was displayed as A and which as B (`display_a_condition`, `display_b_condition`)
- The rater's dimensional ratings for each displayed conversation
- The rater's forced-choice guess (A or B was the 33K version)
- The rater's confidence (1-5)
- Whether the guess was correct (`forced_choice_correct`, computed automatically)
- The rater's free-text reasoning and overall preference

### 5.3 Deblinding at Analysis Only

The condition assignment is only unblinded at the analysis stage. During data collection, no human sees the mapping between display position and condition.
